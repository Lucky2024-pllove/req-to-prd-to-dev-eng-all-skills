# Security Policy

## 范围

`requirements-to-prd` 是 **文档型 Agent Skill**，不包含服务端运行时、不托管用户数据。主要风险来自**误提交机密**或**在 Issue/PR 中粘贴真实业务敏感信息**。

## 请勿提交或公开分享

| 类别 | 示例 |
|------|------|
| API / 模型密钥 | 第三方 LLM、内部服务 token |
| 客户与业务机密 | 未脱敏的合同金额、个人身份信息、未公开商业策略 |
| 本地覆盖 | `*.local.md`、`.env`（已在 [.gitignore](.gitignore) 忽略） |

提交前请 `git status` 自检；demo 金样应使用虚构业务场景（如 DailyBill）。

## 报告安全问题

若发现仓库历史中含有效密钥，或技能说明鼓励不安全做法，请 **不要** 在公开 Issue 中粘贴完整机密。通过 GitHub 私密报告（若已启用）或私信维护者，说明路径、影响与建议修复。
